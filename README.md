# Investment Analytics — SQL Project

## Overview
SQL-based portfolio and transaction analysis project simulating a 
real investment firm environment. Built using MySQL to derive 
actionable business insights from 20,000+ investment records.

## Tools Used
- MySQL Workbench
- SQL (Joins, CTEs, Window Functions, Conditional Aggregation)

## Database Schema
| Table        | Records | Description                        |
|--------------|---------|------------------------------------|
| Clients      | 1,000   | Client demographics & risk profile |
| Accounts     | 2,000+  | Linked investment accounts         |
| Transactions | 20,000+ | Buy and Sell transaction records   |

## Business Questions Answered
- What is the firm's total Assets Under Management (AUM)?
- Which clients contribute the most to portfolio value?
- How do Buy vs Sell volumes trend month-over-month?
- How is portfolio value distributed across risk profiles?
- Which cities generate the highest investment value?

## Key Findings
- Identified a **₹1.98M net portfolio outflow** signaling client retention risk
- Top investor alone held **5M+ in AUM**, flagging high concentration risk
- Medium and Low-risk clients each drove **~34% of total portfolio value**
